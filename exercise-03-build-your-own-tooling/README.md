# Exercise 3: Build Your Own Tooling

Level: Intermediate | Time: 60 mins | Focus: Picking one track and building or extending tooling (custom instructions, MCP server, test pipeline, or open brief)

> Track B (MCP server exercises): open the [mcp-exercises/](mcp-exercises/) folder — that is where all the exercise files and starter code live. Start with [mcp-exercises/SETUP.md](mcp-exercises/SETUP.md) if you have not set up the MCP server yet.

## Exercise summary

You have 60 minutes to build or extend something useful. Pick one track. At the end, each team has 2–3 minutes in Show and Tell to demo what they built (no slides; screen share).

Tracks:

- Track A: Extend your custom instructions from Exercise 2. Add more rules, or a prompt library, or test them on another file. Get them ready to use in your real repo.
- Track B: Standards MCP server. The `mcp-server/` folder in this exercise contains a Node.js stdio server (`mcp-stdin-server/`) and a Python HTTP server (`mcp-http-server/`) that expose UKEF coding standards. Open the `mcp-exercises/` folder and work through its exercises to see the server in action, then extend it by adding a rule to the standards files or a new check in the server code.
- Track C: Test generation pipeline. Use Copilot to generate tests for a function from Exercise 1 or your own code. Script or document the steps so your team could repeat it (e.g. "Open file, run this prompt, paste tests here, run tests").
- Track D: Open brief. Pitch your idea to a facilitator first (e.g. "I want to build X so that Y"). Once they agree the scope is doable in 30 minutes, build it. If you are unsure, pick A, B, or C instead.

This mirrors real work: taking what you learned and turning it into something your team can reuse.

## Success criteria

- You have chosen one track and worked on it for the 60 minutes.
- You have something to show: updated instructions, a running MCP server and a small change, a test pipeline doc, or a small tool from your pitch.
- You are ready to do a 2–3 minute demo in Show and Tell (what you built, how it works, what you would do next).

## Getting started

1. Read the four tracks above. Decide as a team (or individually) which one you are doing.
2. If you choose Track D, find a facilitator and pitch your idea in one sentence. Get a yes before you start.
3. Open the playbook for your track and follow the steps. Focus on one concrete outcome (e.g. "instructions file we can paste into our repo" or "MCP server running and one new check").
4. With 5 minutes left, stop and prepare your 2–3 minute demo: what you built, how you use it, what you would do next.

[View playbook →](playbook-copilot.md)

## What you need

- Track A: Your Exercise 2 custom instructions file and the Exercise 1 sample code (or your repo).
- Track B: The `mcp-server/` folder in this exercise (built before the day — see `SETUP-GUIDE.md` section 4), the `mcp-exercises/` folder, and an editor configured to connect to the MCP server (see `mcp-exercises/SETUP.md`).
- Track C: Exercise 1 sample code (or your own code), Copilot, and a doc or script to record the steps.
- Track D: An idea you can pitch in one sentence and build in 60 minutes; facilitator approval before starting.
